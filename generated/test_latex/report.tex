\chapter{INTRODUCTION}


This chapter introduces the internship undertaken at Edutainer in the role of UX Design, Game Design Intern. It covers the scope, objectives, relevance, and background of the internship.


\section{Internship Scope and Objectives}

The internship scope covered:

\begin{itemize}[leftmargin=1.5em]
    \item Understanding UX design principles and user-centered design
    \item Exploring game mechanics and interactive design
    \item Hands-on use of Figma, Unity, and documentation tools
\end{itemize}

\section{Relevance of UX Design, Game Design Internship to Electronics and Communication Engineering}

UX and Game Design are directly relevant to Electronics and Communication Engineering as modern embedded systems require intuitive interfaces. The skills gained here apply directly to HMI design and IoT product development.


\section{Evolution of Practices in the Field}

The field has evolved from basic command-line interfaces to sophisticated touch-based and voice-based user interfaces, driven by rapid advancement in mobile computing and AI.


\section{Current Trends and Technologies}

Current trends include AI-driven UI/UX personalization, AR/VR integration, and data-driven design using tools like Figma, Adobe XD, and Unity.

% ── CHAPTER 3: WORK DONE / METHODOLOGY ────────────────────────────────────────
\chapter{WORK DONE / METHODOLOGY}


This chapter presents the work completed during the internship and the methodology followed.


\section{Overview of Internship Work}

I focused on UX design tasks, structured learning, and technical documentation. Tasks included completing assignments, recording solutions, and maintaining GitHub repositories.


\section{Tools and Technologies Used}

Tools used included:

\begin{itemize}[leftmargin=1.5em]
    \item Figma and Canva for interface design
    \item Unity for game design concepts
    \item GitHub for repository management
    \item AI tools for content refinement
\end{itemize}



\section{Work Process and Methodology}

The process followed an iterative approach: concept understanding, logical breakdown, practical application through examples, and final documentation.


\section{Key Project and Case Study}

I analyzed Stone Story RPG brewing system which is built on combination logic and outcome-driven experimentation. Key characteristics included:

\begin{itemize}[leftmargin=1.5em]
    \item Combinational Design: Different ingredients produce distinct outcomes
    \item Emergent Gameplay: Players discover patterns over time
    \item Implicit Learning: Teaching through feedback rather than tutorials
\end{itemize}



\section{Challenges Faced}

Key challenges included understanding complex UX concepts, managing large content volumes, and ensuring transcription accuracy.


\section{Solutions Implemented}

Challenges were overcome by breaking concepts into smaller units, using AI tools for refinement, and maintaining organized GitHub repositories.

This chapter presents the results and analysis of the work carried out during the internship at Edutainer.




\section{Outcomes of the Work}

The internship produced organized educational materials, notes, and assignments maintained via GitHub. UX design layouts were produced using Figma and Canva.


\section{Learning Outcomes}

Technical skills in UX thinking, design tools, and organized problem-solving were enhanced. Analytical skills were greatly improved through iterative practice.


\section{Analysis}

The structured approach worked well for establishing consistency. GitHub and AI tools greatly increased efficiency. Some limitations were noted in content extraction accuracy.
